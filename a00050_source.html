1 nnp-prune: Prune symmetry functions
2 ===================================
3 
4 @warning
5 Documentation under construction...
6 
7 The tool `nnp-prune` is used to identify and remove symmetry functions that have
8 little effect on the training performance. There are two modes of operation:
9 
10 ### Prune before training
11 
12 Here, symmetry functions are discarded if their range (i.e. maximum - minimum
13 value over the entire data set) is below a provided threshold. The first step
14 is to calculate all symmetry functions with [nnp-scaling](nnp-scaling.md):
15 
16 ```
17 mpirun -np 4 nnp-scaling 500
18 ```
19 
20 This will produce the file `scaling.data` where minimum, maximum, mean and sigma
21 values are stored. Then, in the same directory call for example
22 
23 ```
24 nnp-prune range 1.0E-4
25 ```
26 
27 to eliminate all symmetry functions with range below @f$10^{-4}@f$. The resulting
28 settings file is called `output-prune-range.nn`. It is an exact copy of
29 `input.nn` but all lines with pruned symmetry functions are commented out. If
30 the result is reasonable, rename the file to `input.nn` and start over with the
31 training process (note: do not forget to calculate `scaling.data` again with
32 `nnp-scaling`).
33 
34 ### Prune after training
35 
36 After a neural network potential has been fitted with `nnp-train` it is possible
37 to identify unnecessary symmetry functions with a so-called sensitivity
38 analysis. Here, the derivatives of the output neuron with respect to the input
39 layer neurons, i.e. the symmetry functions, are averaged over the whole data.
40 This analysis is automatically performed by `nnp-dataset` and the results are
41 stored in files like `sensitivity.008.out` (for the oxygen neural network):
42 
43 ```
44 mpirun -np 4 nnp-dataset 1
45 ```
46 
47 With the sensitivity data ready, use the pruning tool to automatically remove
48 symmetry functions with a sensitivity below the threshold:
49 
50 ```
51 nnp-prune sensitivity 0.5
52 ```
53 
54 Here, the threshold is set to 0.5% where 100% is equal to the total sum of all
55 sensitivities. The resulting settings file is called
56 `output-prune-sensitivity.nn` and is again an exact copy of `input.nn`, only
57 with commented pruned symmetry function lines.
