1 NNP configuration: keywords
2 ===========================
3 
4 @warning
5 Documentation under construction...
6 
7 The NNP settings file (usually named `input.nn`) contains the setup of neural
8 networks and symmetry functions. Each line may contain a single keyword with no,
9 one or multiple arguments. Keywords and arguments are separated by at least one
10 whitespace. Lines or part of lines can be commented out with the symbol "#",
11 everything right of "#" will be ignored. The order of keywords is not
12 important, most keywords may only appear once (exceptions: `symfunction_short`
13 and `atom_energy`). Here is the list of available keywords (if no usage
14 information is provided, then the keyword does not support any arguments).
15 
16 General NNP keywords
17 --------------------
18 
19 These keywords provide the basic setup for a neural network potential.
20 Therefore, they are mandatory for most NNP applications and a minimal example
21 setup file would contain at least:
22 
23 ```
24 number_of_elements 1
25 elements H
26 global_hidden_layers_short 2
27 global_nodes_short 10 10
28 global_activation_short t t l
29 cutoff_type 1
30 symfunction_short ...
31 ...
32 ```
33 These commands will set up a neural network potential for hydrogen atoms (2
34 hidden layers with 10 neurons each, hyperbolic tangent activation function) and
35 the cosine cutoff function for all symmetry functions. Of course, further
36 symmetry functions need to be specified by adding multiple lines starting with
37 the `symfunction_short` keyword.

41 ### `number_of_elements`
42 
43 *Usage:* <code><b>number_of_elements</b> <i>integer</i></code>
44 
45 *Example:* <code><b>number_of_elements</b> <i>3</i></code>
46 
47 Defines the number of elements the neural network potential is designed for.
48 
49 ---------------------------
50 
51 ### `elements`
52 
53 *Usage:* <code><b>elements</b> <i>string <...></i></code>
54 
55 *Example:* <code><b>elements</b> <i>O H Zn</i></code>
56 
57 This keyword defines all elements via a list of element symbols. The number
58 of items provided has to be consistent with the argument of the
59 `number_of_elements` keyword. The order of the items is not important,
60 elements are automatically sorted according to their atomic number. The list
61 nnp::ElementMap::knownElements contains a list of recognized element symbols.
62 
63 ---------------------------
64 
65 ### `atom_energy`
66 
67 *Usage:* <code><b>atom_energy</b> <i>string float</i></code>
68 
69 *Example:* <code><b>atom_energy</b> <i>O -74.94518524</i></code>
70 
71 Definition of atomic reference energy. Shifts the total potential energy by
72 the given value for each atom of the specified type. The first argument is the
73 element symbol, the second parameter is the shift energy.

77 ### `global_hidden_layers_short`
78 
79 *Usage:* <code><b>global_hidden_layers_short</b> <i>integer</i></code>
80 
81 *Example:* <code><b>global_hidden_layers_short</b> <i>3</i></code>
82 
83 Sets the number of hidden layers for neural networks of all elements.
84 
85 ---------------------------
86 
87 ### `global_nodes_short`
88 
89 *Usage:* <code><b>global_nodes_short</b> <i>integer <...></i></code>
90 
91 *Example:* <code><b>global_nodes_short</b> <i>15 10 5</i></code>
92 
93 Sets the number of neurons in the hidden layers of neural networks of all
94 elements. The number of integer arguments has to be consistent with the
95 argument of the `global_hidden_layers_short` keyword. Note: The number of
96 input layer neurons is determined by the number of symmetry functions and there
97 is always only a single output neuron.
98 
99 ---------------------------
100 
101 ### `global_activation_short`
102 
103 *Usage:* <code><b>global_activation_short</b> <i>char char <...></i></code>
104 
105 *Example:* <code><b>global_activation_short</b> <i>t t t l</i></code>
106 
107 Sets the activation function per layer for hidden layers and output layers
108 of neural networks of all elements. The number of integer arguments has to
109 be consistent with the argument of the `global_hidden_layers_short` keyword
110 (i.e. number of hidden layers + 1). Activation functions are chosen via
111 single characters from the following table (see also
112 nnp::NeuralNetwork::ActivationFunction).
113 
114 | Character | Activation function type |
115 |:---------:| ------------------------------- |
116 | l | nnp::NeuralNetwork::AF_IDENTITY |
117 | t | nnp::NeuralNetwork::AF_TANH |
118 | s | nnp::NeuralNetwork::AF_LOGISTIC |
119 | p | nnp::NeuralNetwork::AF_SOFTPLUS |

123 ### `normalize_nodes`
124 
125 Activates normalized neural network propagation, i.e. the weighted sum of
126 connected neuron values is divided by the number of incoming connections
127 before the activation function is applied. Thus, the default formula to calculate the neuron @f$y^{k}_{i}@f$ in layer @f$k@f$,
128 @f[
129 y^{k}_{i} = f_a \left( b^{k}_{i} + \sum_{j=1}^{n_l} a^{lk}_{ji} \, y^{l}_{j} \right),
130 @f]
131 is modified according to:
132 @f[
133 y^{k}_{i} = f_a \left( \frac{b^{k}_{i} + \sum_{j=1}^{n_l} a^{lk}_{ji} \, y^{l}_{j}}{n_l} \right).
134 @f]

138 ### `cutoff_type`
139 
140 *Usage:* <code><b>cutoff_type</b> <i>integer &lt;float&gt;></i></code>
141 
142 *Example:* <code><b>cutoff_type</b> <i>2 0.5</i></code>
143 
144 Defines the cutoff function type used for all symmetry functions. The first
145 argument determines the functional form, see nnp::CutoffFunction::CutoffType
146 for all available options. Use one of the following integer numbers to
147 select the cutoff type. The second argument is optional and sets the parameter
148 @f$\alpha@f$. If not provided, the default value is @f$\alpha = 0.0@f$.
149 
150 | Cutoff # | Cutoff type |
151 |:--------:| ----------------------------- |
152 | 0 | nnp::CutoffFunction::CT_HARD |
153 | 1 | nnp::CutoffFunction::CT_COS |
154 | 2 | nnp::CutoffFunction::CT_TANHU |
155 | 3 | nnp::CutoffFunction::CT_TANH |
156 | 4 | nnp::CutoffFunction::CT_EXP |
157 | 5 | nnp::CutoffFunction::CT_POLY1 |
158 | 6 | nnp::CutoffFunction::CT_POLY2 |
159 | 7 | nnp::CutoffFunction::CT_POLY3 |
160 | 8 | nnp::CutoffFunction::CT_POLY4 |

164 ### `center_symmetry_functions`
165 ### `scale_symmetry_functions`
166 ### `scale_symmetry_functions_sigma`
167 
168 Combining these keywords determines how the symmetry functions are scaled
169 before they are used as input for the neural network. See
170 nnp::SymmetryFunction::ScalingType and the following table for allowed
171 combinations:
172 
173 | Keywords present | Scaling type |
174 | -------------------------------------------------------- | ------------------------------------- |
175 | `None` | nnp::SymmetryFunction::ST_NONE |
176 | `scale_symmetry_functions` | nnp::SymmetryFunction::ST_SCALE |
177 | `center_symmetry_functions` | nnp::SymmetryFunction::ST_CENTER |
178 | `scale_symmetry_functions` + `center_symmetry_functions` | nnp::SymmetryFunction::ST_SCALECENTER |
179 | `scale_symmetry_functions_sigma` | nnp::SymmetryFunction::ST_SCALESIGMA |

199 ### `symfunction_short`
200 
201 *Usage:* <code><b>symfunction_short</b> <i>string int ...</i></code>
202 
203 *Example:* <code><b>symfunction_short</b> <i>H 2 H 0.01 0.0 12.0</i></code>
204 
205 *Example:* <code><b>symfunction_short</b> <i>H 3 O H 0.2 -1.0 4.0 12.0</i></code>
206 
207 *Example:* <code><b>symfunction_short</b> <i>O 9 H H 0.1 1.0 8.0 16.0</i></code>
208 
209 *Example:* <code><b>symfunction_short</b> <i>H 12 0.01 0.0 12.0</i></code>
210 
211 *Example:* <code><b>symfunction_short</b> <i>O 13 0.1 0.2 1.0 8.0 16.0</i></code>
212 
213 Defines a symmetry function for a specific element. The first argument is the
214 element symbol, the second sets the the type. The remaining parameters depend on
215 the symmetry function type, follow the links in the right column of the table
216 and look for the detailed description of the class.
217 
218 | Type integer | Symmetry function type |
219 |:------------:| ------------------------------------ |
220 | 2 | nnp::SymmetryFunctionRadial |
221 | 3 | nnp::SymmetryFunctionAngularNarrow |
222 | 9 | nnp::SymmetryFunctionAngularWide |
223 | 12 | nnp::SymmetryFunctionWeightedRadial |
224 | 13 | nnp::SymmetryFunctionWeightedAngular |
